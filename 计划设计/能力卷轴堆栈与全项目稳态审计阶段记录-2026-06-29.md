# 能力卷轴堆栈与全项目稳态审计阶段记录-2026-06-29

## 本阶段目标

- 将首页 `#skills` 四条能力从静态矩阵升级为卷轴堆栈展示。
- 隐藏原 `ACTIVE SYSTEM / xx` 读数卡，避免继续展示暂时缺少内容支撑的说明区。
- 保持现有 Hero、挂绳卡、右下大桨叶、资料库、视频、图库、证书和粒子体系不变。
- 同步沉淀项目计划、测试要求和可复用高级网站制作 skill。

## 已实施改动

- 新增 `src/components/ui/CapabilityScrollStack.vue`。
- 首页 `HomePage.vue` 直接使用现有 `systemNodes` 作为能力堆栈数据源。
- 移除 `activeSystem/currentSystem` 可见状态链，不再渲染 `system-readout`。
- 清理旧 `.systems-map`、`.system-node`、`.system-readout` 样式引用。
- 更新 `CoreMotionParticles` 的内容保护选择器，让中区粒子避让新的能力堆栈卡片。
- 新增 `.capability-scroll-stack` 和 `.capability-scroll-card` 样式体系。

## 视觉与交互设计

- 桌面端采用 sticky 卷轴堆栈：卡片按滚动进入固定层，后层带轻微缩放、错位、旋转和降亮。
- Hover 只使用细光、边框亮度和轻位移，不增加强模块感。
- 平板和手机端降级为轻量层叠列表，不使用强 sticky，避免和页面滚动、导航、挂绳卡冲突。
- `prefers-reduced-motion` 下取消 sticky 和强 transform，只保留静态层叠排版。

## 测试用例

- 构建：`npm run build` 必须通过。
- 静态检查：旧类名 `systems-map/system-node/system-readout` 不应再出现在渲染组件和主样式中。
- 桌面：`/#/#skills` 下四张能力卡有卷轴堆栈层级，无 `ACTIVE SYSTEM` 读数卡。
- 平板：1024x768、834x1194 下堆栈不压导航，无横向溢出。
- 手机：430x932、390x844、375x812、360x740、320x568 下能力卡可读、可滚动、无横向溢出。
- 降级：reduced-motion 下能力卡为静态层叠。
- 全站回归：Video Bay、资料库六个详情页、两个 reel、证书书本、UAV 图库、AI 堆叠展厅、保密档案均可直达。

## 风险处理

- 若桌面 sticky 影响阅读，优先降低堆叠位移和缩放，不增加解释文案。
- 若移动端出现滚动冲突，继续保持静态层叠，不启用惯性或拖拽。
- 若后续四条能力需要详情内容，可新增独立数据面板或资料库关联，不恢复旧读数卡。
- 复杂 CSS 拆分、组件重命名和体积治理记录为后续 P2，不在本阶段激进删除。
