# 首页挂绳证件卡阶段记录 - 2026-06-28

## 目标

- 将首页右侧 FPV 场景照片从旧科技相框升级为更精致、更有互动感的挂绳证件卡。
- 借鉴 React Bits Lanyard 的挂绳与拖拽回弹思路，但保持当前 Vue3 架构和移动端稳定性。

## 实施结果

- 新增 Vue 原生组件 `HeroLanyardCard.vue`。
- 视觉结构包含顶部锚点、双股 SVG 挂绳、金属夹扣、磨砂透明证件套、照片面、底部身份条和背板信息态。
- 首页 `HomePage.vue` 已用新组件替换旧 `hero-portrait` figure。
- 旧 `.hero-portrait` 全局样式已移除，避免新旧头像样式并行。
- 桌面端支持 hover 3D 倾斜、拖拽拉伸、松手回弹、点击露背。
- 手机/触摸端保留静态和轻触反馈，不启用自由拖拽。
- reduced-motion 下关闭非必要动画。

## 测试结果

- `npm run build` 通过。
- Playwright Core + 本机 Chrome 测试通过：
  - 首页路由可打开。
  - 挂绳证件卡可见。
  - 桌面无横向溢出。
  - 证件卡不遮挡 telemetry。
  - hover 更新 3D 倾斜变量。
  - 拖拽更新偏移并在松开后回到原点。
  - 点击可切换露背信息态。
  - 移动端无横向溢出。
  - reduced-motion 下动画关闭。
  - console error = 0，page error = 0，400+ failed response = 0。
- 截图与 QA 结果保存于 `测试截图/hero-lanyard-card-2026-06-28/`。

## 风险与后续

- 首页 Hero 文案较长，移动端证件卡位于首屏下方，需要滚动查看；当前不强行上移，避免压缩主定位文案。
- 后续如果更换头像，应继续保持主体位于照片中上部，避免底部身份条遮挡关键内容。
- 若真机微信仍出现动画压力，可进一步把 `deviceTier=low` 也纳入静态模式。
