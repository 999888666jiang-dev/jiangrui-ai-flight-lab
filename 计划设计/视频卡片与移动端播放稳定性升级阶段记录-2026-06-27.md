# 视频卡片与移动端播放稳定性升级阶段记录 - 2026-06-27

## 本阶段目标
- 解决手机端、微信浏览器、百度浏览器直接加载大体积 MP4 时出现的黑屏、`NotSupportedError`、`AbortError` 和起播缓慢问题。
- 将所有视频卡片升级为“封面图先行 + 桌面 3 秒 hover 预览 + 点击后 preview 播放 + 手动高清 full”的统一媒体体验。
- 保持 Video Bay 背景/封面视频不参与本次卡片预览改造。

## 已实施内容
- 媒体变体脚本新增 `teaser.mp4` 生成：3 秒、静音、H.264 Baseline、低码率、`faststart`，用于桌面 hover 预览。
- 媒体 manifest 新增 `teaserAsset` 与 `releaseTeaserAsset`，同时保留 `poster`、`preview`、`full` 四层资源。
- 新增统一视频缩略卡组件 `VideoPreviewTile`：
  - 默认只显示 poster。
  - 桌面 hover/focus 时临时挂载 teaser。
  - 鼠标离开后释放 teaser，移动端不预加载 teaser。
- 证据库入口中，FPV 和成交成果展示卡片随机抽取对应素材作为真实封面与 hover 预览来源；其它无真实视频素材的卡片保持设计占位。
- FPV 回忆胶卷与成交成果展柜页面所有卡片接入真实 poster 和 teaser。
- 播放器仍默认使用 preview，只有点击“查看高清完整版本”才切换 full，并保留下载进度条式按钮反馈。
- GitHub Release `media-current` 已上传 87 个轻量资源：poster、preview、teaser。

  - 成交成果 reel 卡片数量：17，poster 数量：17。
  - 成交成果桌面 hover 成功挂载 `.teaser.` 视频。
  - 移动端打开 FPV reel 时，点击前没有 full 请求，没有 teaser 预加载。
  - 移动端点击卡片后，lightbox 默认播放 `.preview.`。
  - console error = 0，page error = 0，400+ failed response = 0。

## 后续规则
- 后续新增视频必须先运行媒体变体生成脚本，再上传轻量资源。
- 移动端默认不得加载原始大视频；full 只能由用户主动点击高清按钮后加载。
- CDN 完成实名认证后，poster、teaser、preview 应优先迁移到腾讯云 COS/CDN，GitHub Release 继续作为 fallback。
