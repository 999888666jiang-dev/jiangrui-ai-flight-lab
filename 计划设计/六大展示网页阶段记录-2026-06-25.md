# 六大展示网页阶段记录 - 2026-06-25

## 背景

用户要求将 Evidence Vault 图片中展示的六个资源卡制作成六个独立展示网页，并通过点击卡片后的“吸入式”动效进入新页面。六个页面作为后续素材和成果展示库，当前先保留高级占位，不伪造未提供素材，不请求不存在资源。

## 已实现范围

### 入口与路由

- `/#/evidence-vault/fpv-flight-video`：FPV 飞行视频。
- `/#/evidence-vault/ai-design-works`：AI 设计作品。
- `/#/evidence-vault/uav-platform-gallery`：全机型无人机图库。
- `/#/evidence-vault/code-projects`：代码与项目。
- `/#/evidence-vault/flight-logs-test-reports`：飞行日志与测试报告。
- `/#/evidence-vault/certificates-awards`：证书与荣誉。

### 关键组件

- `src/data/siteContent.ts`
  - 扩展 `evidenceItems` 数据结构，加入 `slug`、`theme`、`detailLayout`、`metrics`、`assetSlots` 等字段。
  - 六个展示页的真实素材位置全部以数据驱动保留，当前不写入不存在的封面、视频、PDF 或链接。

- `src/components/effects/VaultPortalOverlay.vue`
  - 根据点击卡片的屏幕位置生成吸入式进入动效。
  - 动效包含卡片收缩、HUD 粒子内卷、中心跃迁点和扫描黑场。
  - 在 `prefers-reduced-motion` 下跳过强动效，直接进入路由。

- `src/pages/ShowcaseDetailPage.vue`
  - 用一个详情页组件承载六个 slug。
  - 页面内容数据驱动，但六个 slug 使用不同布局、色彩、动效和交互隐喻，避免模板化。

- `src/pages/EvidenceVaultPage.vue`
  - 六张资源卡升级为键盘可访问入口。
  - 支持点击、Enter、Space 触发进入。
  - 返回证据库后可继续滚动和操作。

## 六个页面设计方向

| 页面 | 主题 | 设计与动效方向 |
| --- | --- | --- |
| FPV 飞行视频 | Velocity Cinema | 视频舱占位、速度线、姿态地平线、REC HUD、飞行片段槽位 |
| AI 设计作品 | Neural Atelier | 神经节点画布、作品流占位、AI 工作流与原型流程 |
| 全机型无人机图库 | Hangar Matrix | 机库网格、平台分类、照片墙槽位、机型资料位 |
| 代码与项目 | Code Ops Console | 终端启动感、电路线、项目卡、仓库和部署状态位 |
| 飞行日志与测试报告 | Black Box Recorder | 时间线、波形扫描、报告 PDF 位、问题复盘模块 |
| 证书与荣誉 | Verification Vault | 证书扫描、核验时间线、荣誉材料槽位 |

## 资源策略

- 当前不使用临时外链、AI 伪造素材或参考视频冒充本人素材。
- 封面、视频、图片、PDF、项目链接全部等用户后续提供真实资源后再接入。
- 资源未提供时只显示页面内的高级占位和“待补充”状态。
- 不渲染缺失文件路径，因此不会因为待补充素材产生 404。
- 六个入口 slug 固定，后续补资源只改数据和资源路径，不改路由结构。

## 风险与处理

| 风险 | 处理 |
| --- | --- |
| 六页变成换文案模板 | 用同一数据组件承载，但每个 slug 配不同主题、局部动效、舞台结构和指标呈现 |
| 吸入动效遮挡可读性或拖慢交互 | 动效只在进入瞬间播放，详情页不强制反向吸入；reduced-motion 直接路由 |
| 缺素材导致页面显空 | 用设计占位承载资源槽位，同时明确“待补充”状态，不伪造内容 |
| 移动端中文标题裁切 | 调整展示页 H1 在窄屏下的换行策略，保留视觉冲击但允许正常换行 |
| 后续接入资源产生 404 | 接入前确认真实文件或地址存在；未确认前不写入数据 |
