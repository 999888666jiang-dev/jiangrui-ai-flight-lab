# Vue3 迁移阶段记录 - 2026-06-25

## 本阶段目标

将现有 `Vite + React + TypeScript` 个人网站迁移为 `Vite + Vue 3 + TypeScript + Vue Router`，保留 AI Flight Lab 的高级动效方向，并把首页长滚动体验扩展为“首页 + 独立子页面”的混合结构。

## 已完成实施

### 框架迁移

- 已改为 `vue`、`vue-router`、`@vitejs/plugin-vue`、`vue-tsc`。
- 已移除 React、React DOM、React Router、React Three Fiber、Drei 与 React Vite 插件依赖。
- `index.html` 改为挂载 `#app` 并加载 `/src/main.ts`。
- `vite.config.ts` 改为 Vue 插件，并保留 `gsap`、`three` 手动分包。
- `tsconfig.app.json` 已改为 Vue SFC 与 TypeScript 路径。

### 路由结构

- `/#/`：首页，保留 Hero、任务、能力系统、Mission Control、联系入口。
- `/#/video-bay`：独立视频舱页面，预留真实视频背景位。
- `/#/evidence-vault`：独立证据库页面，集中展示执照、二维码、简历 PDF 和后续素材位。
- 使用 `createWebHashHistory()`，本地和静态部署不依赖服务器重写。

### Vue 组件结构

- `src/App.vue`：全局路由出口和扫描转场。
- `src/layouts/FlightLayout.vue`：顶部导航、任务侧轨、全局粒子层、鼠标光效。
- `src/pages/HomePage.vue`：长滚动沉浸首页。
- `src/pages/VideoBayPage.vue`：视频背景子页面。
- `src/pages/EvidenceVaultPage.vue`：证据库子页面。
- `src/components/three/FlightHeroScene.vue`：原生 Three.js 透明科技桨叶。
- `src/components/effects/ParticleField.vue`：按路由 preset 切换的粒子背景。

### 动效与交互

- 首页首屏保留透明科技感四叶桨、星场、HUD、鼠标光效。
- 首页区块使用 GSAP ScrollTrigger 渐入。
- 能力矩阵支持点击切换当前读数。
- Mission Control 支持点击切换任务详情。
- Video Bay 在视频缺失时只显示设计占位，不请求缺失视频文件。
- Evidence Vault 保留执照、二维码、PDF 入口和后续资源位。
- 页面切换使用 Vue Router + 遮罩扫描转场。
- 移动端导航改为横向可滑动任务胶囊，降低 3D 成本但保留粒子和 HUD 视觉。

## 资源策略

- 视频目标路径固定为：`public/videos/fpv-lab-background.mp4`。
- 当前不创建假视频、不使用过期临时链接、不主动请求不存在的视频文件。
- 用户后续补充视频后，再把 `VideoBayPage.vue` 的 `videoSrc` 接入真实路径。

## 已修复问题

- Vue `<Transition>` 子页面根节点警告：通过给首页增加单根节点修复。
- 移动端导航不可达：改为固定顶部的横向滑动任务胶囊。
- 移动端证据库标题过大和断字不稳定：收紧 route H1 字号，并将中文标题改为更短的“证据库：真实能力档案”。

## 后续风险

- `three` 包体较大；若后续移动端或首屏性能不足，可把 `FlightHeroScene` 改为动态导入。
- Video Bay 接入真实视频后，需要压缩、设置封面、检查自动播放和移动端流量成本。
- 后续证据资源增加后，必须继续保持“真实证据优先”，避免把占位区误写成已完成项目。
