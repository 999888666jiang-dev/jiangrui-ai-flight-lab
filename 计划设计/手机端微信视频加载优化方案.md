# 手机端/微信视频加载优化方案

更新时间：2026-06-27

适用项目：姜睿个人网站 / AI Flight Lab

## 1. 背景

昨天真机使用时暴露出一个关键问题：手机端、微信浏览器打开视频页时，直接加载 100MB 到 860MB 的原始 MP4 会非常慢，弱网下容易出现黑屏、等待时间过长、后台仍下载大文件等问题。

本次方案目标是：不牺牲网站高级视觉和视频展示能力，但让手机和微信默认只接触轻量资源。

## 2. 最终融合方案

采用三层媒体分发：

1. 生产主源：腾讯云 COS + CDN。
2. 生产兜底：GitHub Releases `media-current`。
3. 本地开发源：`public/videos/`、`public/media/`、`public/media-variants/`。

采用三类媒体变体：

1. `poster.webp`：首屏和卡片默认只加载图片。
2. `preview.mp4`：720p 或不超过源尺寸，H.264，25fps，约 600-900kbps，手机/微信默认播放源。
3. `full.mp4`：高清完整版本，只在用户点击“查看高清完整版本”后加载。

播放优先级：

```text
CDN preview/full -> GitHub Release preview/full -> 本地开发 preview/full -> 高级占位
```

移动端、微信、弱网、`saveData`：

- 不自动加载 full。
- Reel 卡片只加载 poster。
- 灯箱点击后先加载 preview。
- 高清完整版本必须手动点击。
- 页面离开、灯箱关闭或低性能设备释放 video src。

桌面高性能设备：

- 有 preview 时默认 preview。
- preview 尚未生成时允许回退 full，保证开发阶段仍可查看真实视频。
- 高清按钮仍保留。

## 4. 媒体处理流程

安装 FFmpeg 后执行：

```powershell
.\scripts\prepare-media-variants.ps1
```

脚本行为：

- 扫描 `public/videos/fpv-lab-background.mp4`。
- 扫描 `public/media/fpv-flight-video/`。
- 扫描 `public/media/deal-results-showcase/`。
- 为每条视频生成 poster、preview、full。
- 输出到 `public/media-variants/ai-flight-lab/...`。
- 重写 `src/data/mediaVariantsManifest.ts`。

当前本机尚未检测到 `ffmpeg/ffprobe`，所以脚本会明确失败并提示安装，不会静默生成错误资源。

## 5. CDN 上传流程

准备环境变量：

```powershell
$env:TENCENT_SECRET_ID="..."
$env:TENCENT_SECRET_KEY="..."
$env:VITE_MEDIA_CDN_BASE_URL="https://media.example.com"
```

上传：

```powershell
.\scripts\upload-cdn-media.ps1 -Bucket "<bucket-name>" -Region "ap-guangzhou"
```

脚本行为：

- 从 `public/media-variants/ai-flight-lab/` 读取变体资源。
- 上传到 COS `/ai-flight-lab/...`。
- 如果配置了 CDN Base URL，会先用 HEAD 检查 hash 文件是否已存在，已存在则跳过。
- 写入上传报告到 `测试截图/cdn-media-upload-YYYY-MM-DD/upload-result.json`。

CDN 配置要求：

- 开启 HTTPS。
- 开启音视频加速。
- 支持 Range GETs。
- CORS 允许 `GET, HEAD`。
- CORS 允许请求头 `Range`。
- 暴露 `Accept-Ranges, Content-Length, Content-Range`。
- hash 文件名资源长期缓存。

## 6. GitHub Releases 兜底

现有 `scripts/upload-release-media.ps1` 已扩展：

- 没有媒体变体时，继续上传原始视频作为 full 兜底。
- 检测到 `public/media-variants/ai-flight-lab/` 后，会额外上传稳定命名的变体资产：
  - `<group>__<id>__poster.webp`
  - `<group>__<id>__preview.mp4`
  - `<group>__<id>__full.mp4`

Release URL 使用 `?v=<hash>` 刷新缓存。

## 7. 前端播放规则

Video Bay：

- 默认使用 preview。
- 手机/微信 preview 缺失时显示高级占位，不自动请求 full。
- 点击“查看高清完整版本”后切换 full，并尽量保持播放进度。

FPV / 成交成果主舞台：

- 默认使用 preview。
- 随机切换时回到 preview。
- 高清按钮只控制当前播放器。

FPV / 成交成果 Reel：

- 卡片只加载 poster。
- 点击卡片后灯箱优先播放 preview。
- 高清按钮手动切换 full。
- 关闭灯箱时释放 video。

## 8. 测试用例

构建：

- `npm run build` 必须通过。
- `git ls-files` 中不得出现 `.mp4/.mov/.m4v/.webm`。

脚本：

- 无 FFmpeg 时 `prepare-media-variants.ps1` 必须明确报错。
- 有 FFmpeg 时每条视频生成 poster、preview、full。
- CDN 上传无配置时必须明确报错。
- CDN 上传配置完整时写入 JSON 报告。

桌面：

- Video Bay、FPV 主舞台、成交成果主舞台默认使用 preview。
- preview 尚未生成且本地开发时可回退 full。
- 高清按钮可切换 full。

手机/微信模拟：

- Video Bay 不自动请求 full。
- FPV/成交成果详情页不自动请求 full。
- Reel 首屏不请求 MP4，只请求 poster 或显示占位。
- 点击灯箱后先请求 preview。
- 点击高清按钮后才请求 full。

网络：

- console error = 0。
- page error = 0。
- 400+ failed response = 0。
- 横向溢出 = 0。

CDN：

- 抽查 Video Bay、FPV、成交成果各 3 个 CDN URL 返回 200。
- `Range: bytes=0-999` 返回 206 或等效支持分段。
- 响应头支持 `Accept-Ranges`、`Content-Length`、`Content-Range`。

真机：

- iPhone Safari。
- iPhone 微信。
- Android Chrome。
- Android 微信。
- 每台设备验证 Video Bay、FPV 主舞台、成交成果主舞台、两个 Reel 灯箱、高清切换、后台再进入。

## 9. 风险规避

- FFmpeg 未安装：脚本失败并提示安装，不改 manifest。
- CDN 未配置：生产回退 GitHub Releases，仍不让手机默认加载 full。
- preview 缺失：移动端显示高级占位，不请求大视频。
- full 加载慢：只在用户手动选择后加载，并显示加载状态。
- 微信自动播放失败：保留点击播放遮罩。
- 后台持续下载：移动端/低端设备离开页面或关闭灯箱时释放 video src。
- CDN 缓存旧文件：hash 文件名 + manifest 版本参数。
- Git 仓库膨胀：`public/media-variants/` 和所有视频格式继续忽略，不提交 Git。
