# 全系统适配 P0-P1 阶段记录 - 2026-06-27

## 阶段目标

- 本阶段执行融合方案中的 `P0 适配审计与契约` 与 `P1 必做网页适配层`。
- 保留现有 AI Flight Lab 视觉世界、Three.js 透明科技桨叶、三层 Canvas 粒子、视频舱、证据库、FPV 胶卷页和成交成果展柜页。
- 不在本阶段引入 PWA、HLS/CDN、Mux 或 Capacitor，避免基础适配和外部服务风险混在一起。

## 已落地设计

- 新增统一适配层：`DeviceTier`、`RuntimeShell`、`MotionBudget`、`VideoPolicy`、`WebglPolicy` 与 `AdaptationProfile`。
- 新增环境 composable：统一检测桌面、移动浏览器、微信、PWA、Capacitor、iOS、触摸设备、弱网、低动态偏好和设备等级。
- 三个 Canvas 粒子层接入统一预算：设备等级控制粒子数量、DPR、合成模式、扫描线和帧率。
- Three.js 首屏场景接入 WebGL 分级：微信、低端小屏和 reduced-motion 禁用，平板降级，桌面保持完整。
- 视频播放策略升级：所有核心视频保留静音自动播放，同时加入 iOS/微信/弱网播放失败的点击播放遮罩。
- 样式层新增 `safe-area.css`、`responsive.css`、`motion.css`，在现有主样式之后加载，避免大规模重构。
- Hash Router 内部锚点改为 Vue Router hash 跳转，并增加移动端延迟滚动兜底。

## 维护规则

- 后续新增页面或动效不得自行判断 iOS、微信、低性能设备或 reduced-motion，必须读取统一适配层。
- 新增视频播放器必须使用当前视频策略：默认静音、`playsinline`、微信 X5 属性、播放失败显式兜底。
- 新增 fixed/HUD/灯箱/媒体控制必须使用 safe-area 变量。
- P2 的 PWA、路由预加载、HLS/CDN 只能在 P1 QA 通过后继续执行。

## 风险记录

- Playwright 模拟不能替代 iOS Safari 和微信真机，真机仍需单独验收。
- 当前媒体仍是本地 MP4，弱网大视频加载风险存在；本阶段只做懒加载与兜底，不做转码。
- 非 git 仓库，无法按 implement skill 要求提交 commit；本次以文件变更、构建和截图 QA 作为交付证据。
