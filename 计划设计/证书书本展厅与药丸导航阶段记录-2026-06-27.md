# 证书书本展厅与药丸导航阶段记录 - 2026-06-27

## 本阶段目标

- 将顶部导航升级为 Vue 原生药丸导航，不直接引入 React Bits 的 React 组件。
- 将 `证书与荣誉` 页面从普通验证占位升级为书本翻开式证书展厅。
- 将 `个人信息资源包/证书与荣誉/` 中的 PDF 渲染为公开图片展示，不提供 PDF 下载链接。

## 已完成实现

- 新增 `scripts/prepare-certificate-gallery.ps1`：
  - 扫描资源包证书 PDF。
  - 使用 Poppler `pdfinfo.exe` 和 `pdftoppm.exe` 渲染页面 PNG。
  - 输出到 `public/certificates/`。
  - 生成 `src/data/certificatesManifest.ts`。
  - 中文源路径通过 ASCII 安全构造，避免 PowerShell 5 编码问题。
- 已生成 10 个证书条目，合计 10 张轻量封面图和 11 张高清页面图片；其中 `cert-004` 为 2 页 PDF。
- 新增 `FlightPillNav.vue`：
  - 保留 Vue Router、hash 路由 active 判断和语言切换。
  - 桌面端提供 GSAP 药丸扩散、双层文字滑动和 active 胶囊。
  - 平板端横向滚动，小屏端显示当前页胶囊和弹出菜单。
  - reduced-motion、微信和低性能环境通过 `useEnvironment()` 降级。
- 新增 `CertificateBookGallery.vue`：
  - 每份 PDF 对应一本证书薄册。
  - 点击后进入书本打开态。
  - 多页 PDF 支持左右翻页。
  - 支持 Enter/Space 打开、Escape 关闭、ArrowLeft/ArrowRight 翻页。
- `证据库` 中 `证书与荣誉` 入口已接入真实证书封面随机预览。

## 风险处理

- PDF 源文件不进入 `public/`，页面只展示渲染后的 PNG。
- 不提供 PDF 下载入口，降低公开泄露源文件风险。
- Poppler 对中文用户名路径输出的 `nameToUnicode` 警告已在脚本中按退出码处理，非阻断 stderr 不再中断流程。
- 图片路径从 `public/` 根目录剥离，manifest 中只保留 `certificates/...`，适配 GitHub Pages 的 `BASE_URL`。

## 测试要求

- `npm run build` 必须通过。
- `git ls-files` 不得包含证书源 PDF。
- 证书页直达 `/#/evidence-vault/certificates-awards` 必须可见。
- 证书图片请求必须返回 200。
- 桌面端导航 hover、active、点击跳转正常。
- 390x844、430x932、730x945 下导航不遮挡、不横向溢出。
- 证书打开态在桌面和手机端都必须清晰可读。
