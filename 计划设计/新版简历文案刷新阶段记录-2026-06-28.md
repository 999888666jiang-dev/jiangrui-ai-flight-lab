## 测试用例

1. 构建测试：运行 `npm run build`，必须通过。
2. 旧文案扫描：源码不得残留旧迁移状态、旧主定位和旧简历路径。
3. 隐私扫描：源码不得出现手机号、具体期望薪资或公开联系方式过度暴露。
4. 简历事实覆盖：页面文案必须覆盖目标岗位、教育经历、个人网站项目、酣睡博士、极飞、CAAC 资质、竞赛荣誉和直播运营经历。
5. 双语测试：中文和英文切换后，Hero、能力系统、时间线、证据库、视频舱、联系区同步变化。
6. 响应式测试：桌面、平板、手机尺寸下无标题裁切、文字重叠、横向溢出或按钮不可点。
7. 资源测试：新版 PDF 返回 200；旧 PDF 不再作为站点引用。

## 风险与处理

- 长中文标题和英文长句可能撑破移动端卡片：通过行高、`text-wrap`、`overflow-wrap` 和短标题策略处理。
- 前端主线变更后可能削弱 UAV-FPV 特色：保留无人机、FPV、极飞和 CAAC 资质作为差异化证据链，而不是删除。
- 隐私信息公开风险：页面正文不写手机号和具体薪资，保留在简历 PDF 内。
- 文案继续散落风险：新增 `pageCopy` 与 `resumeProfile`，后续优先在 `siteContent.ts` 统一维护。
