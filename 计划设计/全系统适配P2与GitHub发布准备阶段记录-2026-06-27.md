## 发布策略

- 本地开发：`import.meta.env.DEV` 下默认启用本地视频，保留当前素材展示体验。
- GitHub Pages 生产：默认不请求未进入仓库的大视频，避免 404 和 GitHub 单文件限制问题。
- 后续正式公开视频：优先把视频上传到 CDN/对象存储/HLS，再在 `showcaseMedia.ts` 中补 `streamUrl`，或为 Video Bay 设置 `VITE_VIDEO_BAY_SRC`。

## 风险与规避

- GitHub 普通仓库不能接收 100MB 以上文件：通过 `.gitignore` 排除大视频，源码和轻量资源先发布。
- GitHub Pages 无法承载大量原始视频：生产环境默认不请求本地 MP4，后续接 CDN/HLS。
- 本地改动自动同步不能在未配置远端和认证时完成：已准备同步脚本；远端创建和 `origin` 配置后可执行。
- PWA 缓存爆配额：Service Worker 永不缓存视频资源。
