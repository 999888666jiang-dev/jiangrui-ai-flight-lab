# 姜睿个人网站 — 具体实施步骤

> 本文件根据《姜睿个人网站-实施方案.md》拆解为可执行步骤。  
> 抖音临时地址处理原则：当前能访问就先作为视觉和内容参考使用；后续不再维护、不再反复更新；一旦过期，立即从参考范围移除，不作为项目阻塞项。

---

## 0. 实施目标

### 0.1 最终交付物

1. 一个可部署到 GitHub Pages 的静态个人作品集网站。
2. 网站包含中英双语内容。
3. 首屏具备暗黑科技风 3D Hero 场景。
4. 内容区包含 About、Skills & Certifications、Experience Timeline、Video Showcase、Contact。
5. 视频素材完成 Web 端压缩、封面提取和懒加载。
6. 移动端、微信内置浏览器、Safari 有明确降级方案。
7. GitHub Actions 自动构建并发布。

### 0.2 实施优先级

| 优先级 | 内容 | 判断标准 |
|---|---|---|
| P0 | 项目可运行、内容完整、可部署 | `npm run build` 成功，GitHub Pages 可访问 |
| P1 | 3D Hero、视频展示、双语切换 | 核心视觉和求职信息完整 |
| P2 | 性能优化、兼容降级、SEO | 移动端和微信可用，Lighthouse 指标达标 |
| P3 | 玻璃碎裂、自定义光标等增强效果 | 不影响主流程，可最后追加 |

## 2. 项目初始化

### 2.1 创建 Vite React TypeScript 项目

执行：

```bash
npm create vite@latest . -- --template react-ts
npm install
```

如果当前目录已经有文件，建议先在临时目录初始化，再把生成文件复制进项目根目录，避免覆盖已有方案文档。

### 2.2 安装核心依赖

安装运行依赖：

```bash
npm install @react-three/fiber@9 @react-three/drei@9 three gsap react-router-dom@7
```

安装开发依赖：

```bash
npm install -D tailwindcss @tailwindcss/vite typescript eslint prettier
```

版本策略：

1. `package.json` 中使用精确版本号，不使用 `^`。
2. 安装完成后提交 `package-lock.json`。
3. 如出现 React 19 与某依赖兼容问题，优先锁定依赖版本，不随意升级整套技术栈。

### 2.3 配置基础脚本

`package.json` 至少保留：

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsc -b && vite build",
    "preview": "vite preview",
    "lint": "eslint ."
  }
}
```

### 2.4 配置 Vite

`vite.config.ts` 目标：

1. 设置 GitHub Pages 所需 `base`。
2. 配置 `@` 指向 `src`。
3. 将 Three、R3F、GSAP 拆成独立 chunk。
4. 接入 Tailwind。

实施要点：

```ts
export default defineConfig({
  base: '/个人网站仓库名/',
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src'),
    },
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          three: ['three'],
          r3f: ['@react-three/fiber', '@react-three/drei'],
          gsap: ['gsap'],
        },
      },
    },
  },
});
```

验收标准：

```bash
npm run dev
npm run build
```

两条命令均能成功执行。

## 5. 视觉基础与全局样式

### 5.1 设计令牌

在 `theme.css` 中定义：

```css
:root {
  --color-bg: #05070b;
  --color-panel: #0d1118;
  --color-text: #f5f7fb;
  --color-muted: #9aa4b2;
  --color-line: rgba(255, 255, 255, 0.12);
  --color-accent: #4fd1c5;
  --color-accent-2: #7c3aed;
}
```

注意：

1. 首屏可以暗黑科技风。
2. 内容区必须保证可读性，不要全程高对比霓虹。
3. 移动端字号、行高、间距优先保证阅读体验。

### 5.2 全局样式

要求：

1. 使用系统字体栈，避免自定义字体加载失败。
2. `html { scroll-behavior: smooth; }`。
3. 设置 `prefers-reduced-motion` 降级动画。
4. 设置图片和视频最大宽度，避免溢出。

### 5.3 响应式断点

基础策略：

| 视口 | 布局 |
|---|---|
| < 768px | 单列布局，3D 降级或降低粒子数量 |
| 768-1199px | 内容区两列，Hero 保留简化 3D |
| >= 1200px | 完整 3D 与宽屏布局 |

## 6. Hero 3D 场景实施

### 6.1 先实现 Hero DOM 层

`HeroSection` 先包含：

1. 姓名：姜睿。
2. 头衔：软件工程本科生 / CAAC 多旋翼教员。
3. 目标方向：无人机测试岗位。
4. CTA：查看经历、下载简历、联系我。
5. CSS 静态背景，作为 3D 加载前和降级时的 fallback。

验收标准：

1. 不加载 Three.js 时，Hero 仍然完整可读。
2. CTA 按钮可点击。
3. 首屏移动端没有文字遮挡。

### 6.2 懒加载 3D 场景

实施方式：

1. `HeroScene` 使用 `React.lazy` 动态加载。
2. `Suspense` fallback 使用 CSS 背景。
3. 微信浏览器、低端设备、用户开启减少动态效果时，不加载 `HeroScene`。

判断条件：

```text
isWeChat = true -> 静态 Hero
prefers-reduced-motion = true -> 静态 Hero
GPU Tier 0/1 -> 静态或简化 Hero
其他情况 -> 加载完整 HeroScene
```

### 6.3 实现 AmbientGrid

步骤：

1. 先用普通 Plane + GridHelper 做出网格地板。
2. 再替换为轻量 ShaderMaterial。
3. 添加远处透明渐隐。
4. 控制线条透明度，避免抢文字层级。

验收标准：

1. 桌面端网格平滑显示。
2. 移动端不出现明显锯齿或闪烁。
3. Shader 失败时可回退为普通 CSS 背景。

### 6.4 实现 FloatingGeometry

步骤：

1. 创建 5-7 个线框几何体。
2. 设置不同初始位置、旋转速度和运动半径。
3. 用 `useFrame` 做轻量旋转和漂浮。
4. 限制几何体数量，避免遮挡文字。

验收标准：

1. 几何体不挡住姓名和 CTA。
2. 帧率无明显下降。
3. 手机端可隐藏部分几何体。

### 6.5 实现 ParticleField

按四步迭代：

1. 静态粒子：使用 `InstancedMesh` 或 `Points` 显示粒子。
2. 基础动画：添加正弦波或缓慢漂浮。
3. 鼠标交互：通过 raycaster 将鼠标位置转换为 3D 坐标。
4. Shader 推斥：将鼠标坐标传给 uniform，控制粒子远离鼠标。

分级参数：

| 设备等级 | 粒子数量 | 效果 |
|---|---:|---|
| Tier 3 | 2500 | 完整 shader + proximity |
| Tier 2 | 800 | 简化 shader |
| Tier 1 | 300 | 无 proximity，仅轻量动画 |
| Tier 0 | 0 | 静态 CSS 背景 |

验收标准：

1. 桌面 Chrome 帧率接近 60fps。
2. 移动端至少 30fps 或自动降级。
3. Shader 编译失败时页面不白屏。

### 6.6 实现滚动过渡

步骤：

1. 用 ScrollTrigger 或自定义 scroll progress 获取 Hero 滚动进度。
2. `useFrame` 读取 progress。
3. 根据 progress 调整相机 z 轴、粒子透明度、几何体扩散程度。
4. 滚出 Hero 80% 后切到低帧率或 `frameloop="demand"`。

验收标准：

1. 滚动时过渡平滑。
2. 不出现滚动卡顿。
3. 离开首屏后 CPU/GPU 占用下降。

## 8. 视频处理步骤

### 8.1 原视频命名

放入 `source-videos/`：

```text
intro_original.mp4
drone1_original.mp4
drone2_original.mp4
douyin-reference.mp4
```

说明：

1. `douyin-reference.mp4` 只作为参考，不一定进入网站。
2. 网站实际视频应使用本人素材。

### 8.2 H.264 输出

命令模板：

```bash
ffmpeg -i source-videos/intro_original.mp4 -vf "scale=-2:720,fps=24" -c:v libx264 -profile:v baseline -level 3.1 -crf 24 -preset medium -movflags +faststart -an public/videos/intro_h264.mp4
```

如果需要保留音频，去掉 `-an`，并加入：

```bash
-c:a aac -b:a 128k
```

### 8.3 VP9 输出

命令模板：

```bash
ffmpeg -i source-videos/intro_original.mp4 -vf "scale=-2:720,fps=24" -c:v libvpx-vp9 -crf 32 -b:v 0 -an public/videos/intro_vp9.webm
```

### 8.4 提取封面

命令模板：

```bash
ffmpeg -ss 00:00:02 -i source-videos/intro_original.mp4 -frames:v 1 -vf "scale=-2:720" public/videos/posters/intro.webp
```

### 8.5 视频验收

| 项目 | 标准 |
|---|---|
| 单个视频体积 | 优先控制在 3MB 内，最高不超过 6MB |
| 分辨率 | 720p 优先，不够小再降到 540p |
| 帧率 | 24fps 或 30fps |
| 格式 | H.264 MP4 + VP9 WebM |
| 加载方式 | `preload="none"`，点击后播放 |

## 9. 兼容与降级

### 9.1 微信内置浏览器

检测：

```ts
const isWeChat = /MicroMessenger/i.test(navigator.userAgent);
```

处理：

1. 不加载完整 WebGL Hero。
2. 使用静态 Hero 背景。
3. 视频点击后再加载。
4. 顶部显示提示：建议用系统浏览器打开以获得完整体验。

验收标准：

1. 微信里页面可阅读。
2. 不出现白屏。
3. 视频不可自动播放时有可点击入口。

### 9.2 iOS Safari

处理：

1. 视频全部加 `playsInline`。
2. 禁止依赖自动播放。
3. Shader 使用保守语法，或 Safari 直接降级。
4. 避免复杂 `backdrop-filter` 叠加。

### 9.3 低端设备

处理：

1. 降低粒子数量。
2. 禁用 proximity shader。
3. 滚出首屏后暂停渲染。
4. 支持 `prefers-reduced-motion`。

## 10. 性能优化

### 10.1 首屏加载

必须做到：

1. Three.js 相关代码懒加载。
2. 视频不参与首屏加载。
3. 首屏文字、按钮、静态背景优先显示。
4. 图片使用 WebP 或压缩后的 JPG/PNG。

### 10.2 Bundle 检查

执行：

```bash
npm run build
```

检查点：

1. `three` 是否被拆分到独立 chunk。
2. 首屏主包是否明显过大。
3. 没有把原始视频打进 bundle。

### 10.3 运行时优化

1. 进入后台暂停 3D 渲染。
2. 离开 Hero 后降低渲染频率。
3. 视频离开视口后暂停。
4. 只保留一个活动视频。
5. 页面销毁时 dispose Three.js 资源。

## 15. 风险控制执行表

| 风险 | 执行动作 | 触发时机 | 验收 |
|---|---|---|---|
| 抖音链接过期 | 可用则一次性保存；过期则移除参考 | 项目开始当天 | 不阻塞项目 |
| 素材缺失 | 用文字卡片、抽象图形、视频帧替代 | 内容开发前 | 页面无空白区域 |
| Shader 失败 | 降级到普通材质或静态背景 | Hero 开发时 | 不白屏 |
| 首屏过大 | Three 懒加载、视频懒加载 | 构建检查时 | 首屏可快速显示 |
| 微信兼容 | 禁用完整 3D，使用静态背景 | UA 检测为微信 | 内容可读 |
| iOS 视频限制 | 点击后播放，不自动播放 | 视频模块开发时 | iPhone 可播放 |
| 视频过大 | H.264 + VP9 压缩 | 视频上线前 | 单文件体积达标 |
| GitHub Pages 慢 | 接受 MVP，后续可加域名/CDN | 线上测试后 | 可访问即可 |
