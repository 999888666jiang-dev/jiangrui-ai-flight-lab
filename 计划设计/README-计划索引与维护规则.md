# 计划设计索引与维护规则

## 文件夹用途

`计划设计/` 是本项目唯一的计划与测试沉淀目录。以后所有新增计划、旧计划整理、测试用例、测试报告、风险清单、阶段复盘和后续对话产生的执行方案，都统一存放到这个文件夹。

## 当前文件结构

- `项目总计划.md`：整合项目上下文、当前状态、总体目标、阶段路线和更新规则。
- `分支计划-内容资源.md`：简历、照片、证书、微信二维码、视频、后续资料的接入计划。
- `分支计划-视觉交互.md`：页面设计方向、参考风格转译、Three.js 与 CSS 交互计划。
- `分支计划-测试验收.md`：测试范围、用例分类、验收标准和回归策略。
- `测试用例与执行报告-2026-06-25.md`：本轮测试用例设计和实际执行结果。
- `终极优化阶段记录-2026-06-25.md`：AI Flight Lab 终极优化第一轮落地记录。
- `Vue3迁移阶段记录-2026-06-25.md`：React 到 Vue3 迁移、独立子页面、动效系统和回归 QA 记录。
- `侧轨滚动粒子动效阶段记录-2026-06-25.md`：两侧滚轮联动粒子、响应式降级和回归 QA 记录。
- `中区精密粒子动效阶段记录-2026-06-25.md`：中景飞控核心粒子、内容避让、二维码保护和回归 QA 记录。
- `六大展示网页阶段记录-2026-06-25.md`：六个素材展示页、吸入式进入动效、资源占位策略和回归 QA 记录。
- `FPV与成交成果媒体展示阶段记录-2026-06-26.md`：FPV 与成交成果真实视频接入、主舞台播放器、媒体集合页和回归 QA 记录。
- `姜睿个人网站-实施方案.md`：早期实施方案归档。
- `姜睿个人网站-具体实施步骤.md`：早期具体步骤归档。
- `姜睿个人网站-创新升级流程任务方案.md`：AI Flight Lab 升级方案归档。

## 后续维护规则

1. 每次新增需求后，先更新 `项目总计划.md` 的“最近更新”和“当前状态”。
2. 涉及内容、资源、视觉、交互、测试、部署时，同步更新对应分支计划。
3. 每次测试后新增一份 `测试用例与执行报告-YYYY-MM-DD.md`，并在 `分支计划-测试验收.md` 更新最新测试摘要。
4. 用户后续补充的新资源仍先进入 `个人信息资源包/`，确认用途后再接入 `public/` 或数据文件。
5. 不再使用过期抖音临时地址作为页面参考或资源来源。
6. 不虚构飞行时长、项目结果、商业成绩；未提供资源的位置保持“待接入”状态。
