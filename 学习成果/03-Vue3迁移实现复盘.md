# Vue3 迁移实现复盘 - 2026-06-25

## 可复用结论

- 高级动效网站迁移框架时，先锁定视觉世界观和交互结构，再迁移组件；否则容易迁成普通模板页。
- 对个人作品站，Hash Router 更适合本地预览和静态部署，能减少服务器 rewrite 风险。
- 原生 Three.js 可以替代 React Three Fiber，但必须把生命周期绑定到 Vue 的 `onMounted` 和 `onUnmounted`，避免画布、监听器和动画帧泄漏。
- 粒子层适合做成全局 layout 组件，并按 route preset 切换，而不是每个页面重复实现。
- 视频资源缺失时，应显示设计占位并避免请求不存在文件；高级感来自清楚的状态，而不是假素材。
- 移动端不是隐藏导航，而是重排成可操作入口。本轮改为横向胶囊导航后，Video Bay 和 Evidence Vault 在手机上也可达。

## 验证方式

- `npm run build` 通过。
- Playwright Core + 本机 Chrome 完成桌面和移动端回归。
- 覆盖首页、能力矩阵、Mission Control、语言切换、联系区、Video Bay、Evidence Vault。
- 浏览器 console warning、page error、400+ failed response 均为 0。

## 后续迁移建议

- 如果 Three.js 包体继续增长，优先把 `FlightHeroScene.vue` 动态导入。
- 如果接入真实视频，必须补充海报图、压缩版本、移动端轻量替代和自动播放验证。
- 如果证据库资源增加，优先增加真实状态和筛选逻辑，再增加视觉动效。
