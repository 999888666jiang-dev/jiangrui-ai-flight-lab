# 终极优化实现复盘

本次优化将学习阶段的方法实际迁移到姜睿网站，形成当前版本的 `AI Flight Lab / Mission Control`。

## 已验证有效的迁移

- Alche 式暗场光学实验室：转译为透明科技感四叶桨、网格、HUD、粒子和光学暗场。
- Cappen 式字体冲击：转译为大字号任务标题、Mission Control 窗口和编号系统。
- byChudy 式可操作系统：转译为能力矩阵点击、任务窗口切换，不复制桌面系统。
- TWICE 式媒体舞台：转译为独立 Video Bay，用于后续真实 FPV/无人机视频背景。
- JOSEPH 式证据清晰度：保留证书、二维码、PDF 的清晰展示，不让动效遮挡可信资源。
- Lisovskiy 式作品档案：保留真实时间线和资源依据，不虚构飞行时长或项目成果。
- Thingy & Thingy 式人格强度：提升首屏记忆点和网站态度，但保留简历站专业可信度。

## 实现策略

- 不新增依赖，沿用 React/Vite/Three.js/GSAP。
- 通过 CSS 变量实现指针光效与全局氛围层。
- 通过固定任务侧轨建立浏览路径和滚动进度。
- 通过 React state 实现能力节点和任务窗口交互。
- 视频舱不主动请求缺失素材，避免 404；等用户补充视频后再接入。
- 移动端隐藏侧轨、重排视频舱和任务窗口，避免桌面版硬缩放。

## 可继续沉淀到 skill 的经验

- 高级网站落地时，先把参考站思路翻译成当前项目世界观，再写组件。
- 交互不需要多，但必须改变信息层级：节点切换、窗口切换、视频舞台、锚点直达都比普通 hover 更有价值。
- 视频背景位在无素材阶段应显示高级占位，不应使用临时链接或伪造素材。
- hash 直达在单页应用中需要额外兜底，尤其是移动端和动态高度页面。
- QA 不只看 build，必须截图检查首屏、交互态、视频区和移动端。
