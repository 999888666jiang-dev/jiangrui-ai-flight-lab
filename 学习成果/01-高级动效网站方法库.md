# 高级动效美感网站方法库

本方法库记录本项目中已经确认可行、可迁移、值得后续继续使用的网页制作知识。后续每次优化姜睿个人网站时，应把新验证过的做法追加到这里，并同步沉淀进 `高级动效美感网站-skill/SKILL.md`。

## 1. 先定“世界观”，再定组件

高端创意网站通常不是从卡片、按钮、网格开始，而是先确定一种世界观：暗场实验室、桌面系统、媒体舞台、奢侈品橱窗、字体剧场、编辑画册、反广告海报。组件只是世界观的延伸。姜睿网站的世界观应是“AI Flight Lab / 飞行实验室”，组件应像仪表、任务卡、飞行日志、资源舱，而不是普通简历模块。

实施方法：

- 首屏必须有唯一主视觉，例如透明科技感桨叶、航线光轨、AI 数据层。
- 所有按钮、卡片、标签、进度条都围绕飞行/AI/HUD 语言设计。
- 不使用泛化的“关于我、技能、项目”视觉模板，改成“身份校准、飞行能力、任务案例、资源证据、连接通道”。

## 2. 动效要服务空间，不服务炫技

动效的价值是告诉用户“现在进入了哪个区域、哪个对象可操作、点击后发生了什么”。如果动效不改变信息关系，只是闪烁、漂浮、放大，就会显得廉价。

可用动效层级：

- 入场动效：首屏主视觉从低亮度唤醒，文字分层出现。
- 滚动动效：章节推进时，背景网格、坐标、主视觉状态同步变化。
- 悬停动效：项目卡显示航线、参数、视频/图片预览。
- 点击动效：任务卡像控制面板展开，而不是简单跳转。
- 状态动效：筛选 AI / FPV / 程序 / 证书时，页面颜色或仪表读数变化。
- 退出动效：弹窗关闭、菜单收起要有方向感，避免突然消失。

## 3. 高级感来自克制的复杂度

参考站的共同点不是都很复杂，而是复杂度集中在一个地方。Alche 把复杂度放在 3D/光影；Joseph 把复杂度放在真实商品图和细节留白；Cappen 把复杂度放在字体运动；byChudy 把复杂度放在可点击窗口系统。一个页面如果同时堆 3D、粒子、渐变、弹窗、跑马灯、旋转卡片，会失去判断力。

姜睿网站建议复杂度分配：

- 最高复杂度：首屏透明科技桨叶与飞行 HUD。
- 中等复杂度：能力矩阵与项目任务卡。
- 低复杂度：证书、二维码、照片、联系方式，保持清晰可信。

## 4. 视觉系统建议

颜色：

- 主背景以深色冷调为主，但避免纯黑大面积压迫。
- 主色可使用冰蓝、青绿、冷白，作为科技/HUD 层。
- 辅色可用少量警示橙或红，用于飞行状态、风险、关键 CTA。
- 避免整站紫色渐变、模板化蓝紫光晕、无意义玻璃卡堆叠。

字体：

- 中文正文需要易读，标题可用更机械、更窄、更技术的字重。
- 英文缩写和参数可作为 HUD 字符层，但不要让正文全变成装饰字符。
- 大标题不要每屏都巨大，只有首屏和关键章节使用高冲击字号。

布局：

- 首屏可以采用主视觉全屏舞台，文字和参数围绕它布置。
- 能力模块采用坐标/雷达/矩阵布局，比普通四宫格更贴合身份。
- 项目模块采用任务卡或飞行记录，不用普通作品集卡片。
- 资源模块必须清晰，用户照片、执照、二维码不能被动效遮挡。

## 5. 滚动叙事模板

推荐五段式：

1. 起飞：主视觉唤醒，说明“姜睿是谁”。
2. 导航：AI 设计、无人机、FPV、程序能力作为航线节点。
3. 执行：展示具体项目或案例，卡片可展开为任务面板。
4. 证据：展示执照、照片、视频、二维码、简历 PDF，占位明确。
5. 降落：联系入口、下载简历、微信二维码、补充资源。

每段滚动应有一个明确状态变化，不能只是文字上移。

## 6. 可迁移交互模式

- 桌面隐喻：适合把个人资料、项目、证书做成可拖拽或可展开窗口，但要控制数量。
- 侧边任务栏：适合快速跳转能力区，兼具进度指示。
- 全屏滑屏：适合展示强视觉作品或无人机影像，但正文信息不宜太多。
- 项目索引：适合大量案例，左侧为项目名，右侧 hover/点击显示画面。
- 媒体舞台：适合 FPV 视频和无人机作品，视频失败时必须有封面图兜底。
- 反常规文案：可以在少量位置增加个性，但简历站需要保持专业可信。

## 7. 技术实现建议

基础栈：

- React/Vite 项目继续沿用现有结构。
- 主视觉可用 Three.js / React Three Fiber 或 Canvas；若性能压力过大，先用 CSS 3D + SVG/Canvas 组合实现。
- 滚动动效可用 GSAP ScrollTrigger；轻量交互可用 Framer Motion 或 CSS transition。
- 大图、视频、PDF、证书全部放入明确资源目录，并做 loading/error fallback。

性能守则：

- 动画优先 transform/opacity，避免频繁触发布局。
- WebGL 场景限制 draw calls、纹理尺寸和 post-processing。
- 移动端减少透明叠层和模糊滤镜。
- 首屏视频必须可替换成海报图。
- 所有动效提供 `prefers-reduced-motion` 降级。

测试守则：

- 每次大改后运行构建。
- 用 Playwright 截桌面和移动端关键帧。
- 检查 console error、网络失败、图片是否实际加载。
- 交互模块至少测试 hover、click、keyboard focus、关闭和重复打开。
- 截图对比不能只看首屏，要覆盖滚动中段和底部。

## 8. 禁止直接照搬

不能复制参考站的文案、品牌图形、专属布局、商业素材、人物图、作品图。可学习的是结构方法：如何建立世界观、如何让滚动像镜头、如何让点击改变空间层级、如何用限制形成辨识度。

对姜睿网站而言，最关键的是把参考站方法翻译成个人能力的表达：飞行、AI、程序、无人机资质和真实资源，而不是把创意 agency、明星站或电商站的形式原样搬过来。
