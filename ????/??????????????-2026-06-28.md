# 首页挂绳顶部贯穿修正阶段记录 - 2026-06-28

## 触发原因

- 用户反馈当前挂绳仍不符合 React Bits Lanyard 参考效果，明确要求绳子或织带顶到页面最顶上。
- 用户常用预览宽度约 730-897px；旧交互在该宽度下被全站适配层判定为移动浏览器，桌面鼠标拖拽没有触发。

## 实施结果

- 保留 Vue3 原生实现，不重新引入 React、React Three Fiber、Rapier、meshline 或 glb 模型。
- `HeroLanyardCard.vue` 新增织带几何计算：组件挂载、resize、scroll 时根据证件卡位置实时更新 `--strap-top-offset` 与 `--strap-length`，让黑色织带从视口顶端垂下，而不是只挂在组件内部。
- `Hero` 区域纵向溢出改为可见，避免长织带被父级裁切；横向溢出仍由全站 `overflow-x: clip` 兜底。
- 681-1080px 桌面窄窗口下，证件卡上移到首屏可见区域，打开页面即可看到从顶部垂挂下来的证件卡。
- 拖拽判定改为组件级 fine pointer 检测：使用 `(hover: hover) and (pointer: fine)`，避免桌面窄窗口被 `navigator.maxTouchPoints` 误判为触摸端后失去拖拽。
- 黑色织带标记从 3 组增加到 6 组，长织带状态下更接近真实 Lanyard 布带比例。

## 测试结果

- `npm run build` 通过；`three` chunk 体积提示继续按既有非阻断风险记录。
- Playwright Core + 本机 Chrome 复测通过：
  - 1440x1000：织带从视口顶端垂下，卡片首屏可见，导航不被遮挡。
  - 897x945：织带顶端到达视口顶部，卡片首屏可见，无横向溢出。
  - 730x945：桌面窄窗口拖拽已启用，`aria-label` 为 `Drag or click to inspect flight pass`。
  - 730x945 拖拽中：`--pass-x = 90px`、`--pass-y ≈ 106px`、`--chain-stretch ≈ 1.189`。
  - 松手回弹后：`--pass-x = 0px`、`--pass-y = 0px`、`--chain-stretch = 1`。
  - 390x844：无横向溢出，触摸端仍不启用自由拖拽。
  - console error = 0，page error = 0。
- 新截图保存于 `测试截图/hero-lanyard-top-2026-06-28/`。

## 后续风险

- 织带从视口顶端贯穿后，在窄桌面宽度会有少量覆盖 Hero 文案的情况；当前保留这种真实挂件遮挡感，因为用户明确表示内容可通过手动拖开查看。
- 手机端仍不启用自由拖拽，避免与页面滚动冲突；如后续需要手机也可移开卡片，应设计独立“收起/移开”按钮，而不是直接启用触摸拖拽。
